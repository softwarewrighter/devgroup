The AI agents repeatedly violate this rule, admit that they should not have.  I want to actually prevent this behavior.  I need a way to prevent write access to anything other than one local cloned repo under delelopment.  This is a serious problem and I want it to stop.

You need hard enforcement, not more prompt wording.

The right design is:

One writable worktree per agent

Everything else mounted read-only

No Git credentials that can push elsewhere

OS/container sandboxing so the agent physically cannot write outside its lane

A promotion path for changes that need to move between repos

That shifts this from “please obey” to capability-based control.

What is going wrong
Your current setup probably has one or more of these failure modes:

the agent can see sibling repos through the filesystem

those repos are writable by the same Unix user

.git metadata and credentials allow commits/pushes in multiple places

shared dependency directories are writable

symlinks or relative paths like ../../repo-b let the agent escape

the agent is running as you, so it inherits all your normal permissions

Once that is true, the agent will eventually “helpfully” fix the other repo. Prompt constraints are weak compared to filesystem permissions.

What to do instead
Recommended architecture
Use per-agent sandboxes with this policy:

Writable
exactly one repo working copy

one temp directory

optional cache directories you explicitly allow

Read-only
all other repos

dependency mirrors

package caches, if safe

shared documentation inputs

No access
your real ~/.ssh

your real Git credential store

parent directories that allow wandering

Docker socket, unless absolutely required

host root filesystem beyond what is needed

That can be done with:

containers: Docker/Podman

bubblewrap / bwrap

chroot / namespaces

VMs for highest isolation

bind mounts with :ro

separate Unix users with restrictive permissions

For most development cases, Podman or Docker + read-only mounts is the fastest path.

Best practical solution
Option A — Container per agent with one RW repo and all else RO
This is the cleanest balance of safety and usability.

Layout on host
For example:

/work/agents/
  repo-a-rw/         # writable clone for agent A
  repo-b-rw/         # writable clone for agent B
/shared/
  repo-a-mirror/     # optional read-only mirror
  repo-b-mirror/     # optional read-only mirror
/cache/
  cargo/
  npm/
  tmp/
Container policy for agent working on repo A
Mount:

/workspace/repo -> /work/agents/repo-a-rw as rw

/deps/repo-b -> /shared/repo-b-mirror as ro

/tmp/agent -> dedicated temp as rw

cargo/npm caches either dedicated or read-only depending on workflow

Do not mount:

your home directory

other writable repos

SSH agent socket unless absolutely needed

Docker socket

GitHub token for other repos

Docker example
docker run --rm -it \
  --name agent-repo-a \
  --user "$(id -u):$(id -g)" \
  --workdir /workspace/repo \
  -v /work/agents/repo-a-rw:/workspace/repo:rw \
  -v /shared/repo-b-mirror:/deps/repo-b:ro \
  -v /cache/cargo-agent-a:/cargo-home:rw \
  -v /tmp/agent-a:/tmp:rw \
  -e CARGO_HOME=/cargo-home \
  -e HOME=/tmp/home \
  --read-only \
  --tmpfs /run \
  --tmpfs /tmp \
  my-agent-image
Then add back only the few writable mounts you actually need.

Why this works
Even if the agent decides:

“I know I was told not to edit repo-b, but I’ll fix it anyway.”

the write simply fails because /deps/repo-b is mounted read-only.

That is the control you want.

Option B — bwrap sandbox on Linux
If you want less overhead than containers, bubblewrap is excellent.

Example concept:

bwrap \
  --bind /work/agents/repo-a-rw /workspace/repo \
  --ro-bind /shared/repo-b-mirror /deps/repo-b \
  --dir /tmp \
  --proc /proc \
  --dev /dev \
  --chdir /workspace/repo \
  /usr/bin/bash
You can further:

use a private home dir

hide /home/mike

hide SSH config

expose only selected binaries and libraries

This is very strong on Linux, especially for local agent harnesses.

Option C — Separate Unix users + filesystem ACLs
This helps, but by itself is usually not enough unless carefully done.

Pattern
create one Unix user per agent, like agent_repo_a

own repo A writable clone by that user

make all other repos owned by different users or root

remove group write permissions

use ACLs to grant read-only where needed

Example:

sudo useradd -m agent_repo_a
sudo chown -R agent_repo_a:agent_repo_a /work/agents/repo-a-rw
sudo chmod -R u+rwX,go-rwx /work/agents/repo-a-rw
sudo setfacl -R -m u:agent_repo_a:rX /shared/repo-b-mirror
sudo setfacl -R -m u:agent_repo_a:0 /work/agents/repo-b-rw
This can work, but it is easier to make mistakes than with container mounts.

Stronger than filesystem permissions: remove the writable path entirely
Use bare mirrors or archives for cross-repo reads
If repo A needs to inspect repo B for dependency resolution, do not expose repo B’s actual writable clone.

Instead expose one of:

a bare mirror

a read-only snapshot

a tarball extraction

a package artifact

a published crate/package/dependency cache

Better pattern
Instead of:

/work/repo-a
/work/repo-b
where both are writable,

use:

/work/agents/repo-a-rw
/shared/snapshots/repo-b-readonly
If repo B changes, refresh the snapshot via a controlled script.

This prevents the agent from “helpfully” editing the source of truth.

Git-level controls
Filesystem isolation is primary. Git controls are secondary but still useful.

1. Remove push credentials from agent environments
Do not let the agent inherit your normal ~/.gitconfig, ~/.ssh, or credential helpers.

Instead:

use a minimal per-agent .gitconfig

no credential helper unless needed

no SSH keys unless needed

if push is needed, give a token that can only access one repo

Example minimal git config
[user]
    name = Agent Repo A
    email = agent-repo-a@example.invalid

[core]
    autocrlf = false

[credential]
    helper =
Set:

export HOME=/tmp/agent-home
export GIT_CONFIG_GLOBAL=/tmp/agent-home/.gitconfig
2. Use repo-scoped tokens
If the agent must push:

give it a token that can only push to repo A

do not give org-wide write scopes

ideally use deploy keys or fine-grained PATs

Then even if it somehow finds another repo, remote push fails.

3. Disable or intercept git push
In many agent runs, push should not happen automatically at all.

You can shadow git with a wrapper:

#!/usr/bin/env bash
if [ "$1" = "push" ]; then
  echo "git push disabled in this sandbox"
  exit 1
fi
exec /usr/bin/git "$@"
This is not a security boundary if the agent can bypass it, but inside a controlled container it is useful belt-and-suspenders.

Avoid common escape hatches
1. Symlinks
If writable repo A contains symlinks pointing outside the sandbox, tools may follow them.

Mitigations:

avoid following symlinks in automation

resolve and inspect repo trees before runs

mount only the minimal subtree

in CI/harness, reject symlinks escaping workspace

2. .. path traversal
If parent directories are mounted writable, the agent can walk upward.

Mitigation:

mount only the repo at /workspace/repo

do not mount /workspace as host parent writable

no writable parent of the worktree

3. Build tools writing outside repo
Cargo, npm, pip, and language servers may write to shared homes/caches.

Mitigation:

set dedicated env vars:

HOME

CARGO_HOME

RUSTUP_HOME

npm_config_cache

XDG_CACHE_HOME

TMPDIR

make them private per agent

4. Editor/IDE helpers
Some agents invoke tools that update lockfiles or format nearby repos.

Mitigation:

don’t expose nearby repos as writable

keep tool roots local to the sandbox

5. Docker socket
If the agent can access /var/run/docker.sock, it can often escape all your controls.

Mitigation:

do not mount Docker socket into the agent sandbox

Promotion-based multi-repo workflow
When repo A discovers repo B needs a change:

issue text

patch file

branch suggestion

failing test reproduction

This mirrors how human teams should work:

one code owner per repo

cross-repo changes are handed off, not directly edited

Artifact examples

This is much better than letting one agent roam.

1. Shared group permissions

Simple sudo script
Here is a straightforward script you can run as root or with sudo.

#!/usr/bin/env bash
set -euo pipefail

# Usage:
#   sudo ./create-dev-users.sh deva1 deva2 devb1 devh1
#
# Behavior:
# - creates shared group "devread" if missing
# - creates each user if missing
# - ensures each user is in devread
# - creates ~/github
# - sets permissions so other devread members can read/traverse
# - owner can write; group cannot write

SHARED_GROUP="devread"

if [[ $# -lt 1 ]]; then
  echo "Usage: sudo $0 user1 [user2 ...]"
  exit 1
fi

if ! getent group "${SHARED_GROUP}" >/dev/null; then
  echo "Creating group: ${SHARED_GROUP}"
  groupadd "${SHARED_GROUP}"
fi

for user_name in "$@"; do
  home_dir="/home/${user_name}"
  github_dir="${home_dir}/github"

  if id "${user_name}" >/dev/null 2>&1; then
    echo "User already exists: ${user_name}"
  else
    echo "Creating user: ${user_name}"
    useradd -m -s /bin/bash "${user_name}"
  fi

  echo "Adding ${user_name} to group ${SHARED_GROUP}"
  usermod -a -G "${SHARED_GROUP}" "${user_name}"

  echo "Creating ${github_dir}"
  mkdir -p "${github_dir}"

  echo "Setting ownership"
  chown -R "${user_name}:${SHARED_GROUP}" "${home_dir}"

  echo "Setting permissions on ${home_dir}"
  chmod 750 "${home_dir}"

  echo "Setting permissions on ${github_dir}"
  chmod 2750 "${github_dir}"

  echo "Done with ${user_name}"
  echo
done

echo "All requested users processed."
echo
echo "Recommended next step:"
echo "  Log out/in for group membership to fully apply to interactive sessions."
Why 2750 on ~/github?
The leading 2 sets the setgid bit on the directory.

That means new subdirectories/files created under ~/github tend to inherit the directory’s group, which is helpful for keeping things readable by the shared group.

So if deva1 clones into:

/home/deva1/github/sw-embed/sw-cor24-apl
the group is more likely to stay devread instead of drifting to deva1.

#!/usr/bin/env bash
set -euo pipefail

# Usage:
#   sudo ./create-dev-users.sh deva1 deva2 devb1 devh1
#
# Creates:
# - shared group: devgroup
# - each user with home dir and bash shell
# - ~/github with shared-read permissions
# - sane .bashrc additions
#
# Permission model:
# - /home/user           => 750
# - /home/user/github    => 2750
# - umask                => 027
#
# Result:
# - owner can write
# - devgroup can read/traverse
# - others have no access

GROUP_NAME="devgroup"

if [[ $# -lt 1 ]]; then
  echo "Usage: sudo $0 user1 [user2 ...]"
  exit 1
fi

if ! getent group "${GROUP_NAME}" >/dev/null; then
  echo "Creating group ${GROUP_NAME}"
  groupadd "${GROUP_NAME}"
fi

for user_name in "$@"; do
  home_dir="/home/${user_name}"
  github_dir="${home_dir}/github"
  bashrc="${home_dir}/.bashrc"

  if id "${user_name}" >/dev/null 2>&1; then
    echo "User exists: ${user_name}"
  else
    echo "Creating user: ${user_name}"
    useradd -m -s /bin/bash "${user_name}"
  fi

  echo "Adding ${user_name} to ${GROUP_NAME}"
  usermod -a -G "${GROUP_NAME}" "${user_name}"

  echo "Creating ${github_dir}"
  mkdir -p "${github_dir}"

  # Home belongs to user, primary group remains user's own group.
  chown "${user_name}:${user_name}" "${home_dir}"
  chmod 750 "${home_dir}"

  # github dir belongs to user:devgroup so cross-dev reads work consistently.
  chown "${user_name}:${GROUP_NAME}" "${github_dir}"
  chmod 2750 "${github_dir}"

  # .bashrc baseline
  touch "${bashrc}"
  chown "${user_name}:${user_name}" "${bashrc}"
  chmod 640 "${bashrc}"

  if ! grep -q '^[[:space:]]*umask[[:space:]]\+027[[:space:]]*$' "${bashrc}"; then

2) validate-dev-users.sh

#!/usr/bin/env bash
set -euo pipefail

GROUP_NAME="devgroup"

if [[ $# -lt 1 ]]; then
  echo "Usage: $0 user1 [user2 ...]"
  exit 1
fi

fail=0

echo "Checking shared group: ${GROUP_NAME}"
if ! getent group "${GROUP_NAME}" >/dev/null; then
  echo "ERROR: group ${GROUP_NAME} does not exist"
  exit 1
fi

echo

for user_name in "$@"; do
  home_dir="/home/${user_name}"
  github_dir="${home_dir}/github"
  bashrc="${home_dir}/.bashrc"

  echo "=== ${user_name} ==="

  if ! id "${user_name}" >/dev/null 2>&1; then
    echo "ERROR: user missing"
    fail=1
    echo
    continue
  fi

  if id -nG "${user_name}" | tr ' ' '\n' | grep -qx "${GROUP_NAME}"; then
    echo "OK: in ${GROUP_NAME}"
  else
    echo "ERROR: not in ${GROUP_NAME}"
    fail=1
  fi

  if [[ -d "${home_dir}" ]]; then
    perms=$(stat -c '%a %U:%G' "${home_dir}")
    echo "HOME:   ${perms}"
  else
    echo "ERROR: home missing"
    fail=1
  fi

  if [[ -d "${github_dir}" ]]; then
    perms=$(stat -c '%a %U:%G' "${github_dir}")
    echo "GITHUB: ${perms}"
  else
    echo "ERROR: github dir missing"
    fail=1
  fi

  if [[ -f "${bashrc}" ]]; then
    perms=$(stat -c '%a %U:%G' "${bashrc}")
    echo "BASHRC: ${perms}"
    if grep -q '^[[:space:]]*umask[[:space:]]\+027[[:space:]]*$' "${bashrc}"; then
      echo "OK: .bashrc sets umask 027"
    else
      echo "WARN: .bashrc missing umask 027"
    fi
  else
    echo "ERROR: .bashrc missing"
    fail=1
  fi

  echo
done

exit "${fail}"

#!/usr/bin/env bash
set -euo pipefail

  echo "Usage: sudo $0 owner_user other_user"
  exit 1

echo

  echo "OK: ${other_user} can read ${probe_file}"

  echo "FAIL: ${other_user} cannot read ${probe_file}"
  exit 2

echo
echo "Testing write denial as ${other_user}"
if sudo -u "${other_user}" bash -lc "echo denied-test > '${probe_dir}/SHOULD_NOT_WRITE.txt'" 2>/dev/null; then
  echo "FAIL: ${other_user} was able to write into ${probe_dir}"
  exit 3
else
  echo "OK: ${other_user} cannot write into ${probe_dir}"
fi

echo
echo "Probe complete."

#!/usr/bin/env bash
set -euo pipefail

# Portable devgroup workspace bootstrap
#
# Run from the root of the checked-out admin repo, for example:
#   cd /mnt/disk2/github/softwarewrighter/devgroup
#   sudo ./scripts/setup-devgroup.sh \
#     --group devgroup \
#     --admin mike \
#     --users deva1,deva2,devb1,devh1
#
# Optional:
#   --init-bare sw-embed/sw-cor24-apl=https://github.com/sw-embed/sw-cor24-apl.git
#   --init-bare sw-embed/sw-cor24-basic=https://github.com/sw-embed/sw-cor24-basic.git

GROUP_NAME="devgroup"
ADMIN_USER="mike"
USERS_CSV=""
declare -a INIT_BARE_SPECS=()

usage() {
  cat <<EOF
Usage:
  sudo $0 --users user1,user2,... [--group devgroup] [--admin mike]
          [--init-bare org/repo=https://github.com/org/repo.git ...]

Options:
  --group       Shared unix group for workers and admin (default: devgroup)
  --admin       Coordinator/admin user (default: mike)
  --users       Comma-separated worker user list (required)
  --init-bare   Optional bare repo spec; may be repeated
  --help        Show this help

Example:
  sudo $0 \
    --group devgroup \
    --admin mike \
    --users deva1,deva2,devb1 \
    --init-bare sw-embed/sw-cor24-apl=https://github.com/sw-embed/sw-cor24-apl.git \
    --init-bare sw-embed/sw-cor24-basic=https://github.com/sw-embed/sw-cor24-basic.git
EOF
}

while [[ $# -gt 0 ]]; do
  case "$1" in
    --group)
      GROUP_NAME="$2"
      shift 2
      ;;
    --admin)
      ADMIN_USER="$2"
      shift 2
      ;;
    --users)
      USERS_CSV="$2"
      shift 2
      ;;
    --init-bare)
      INIT_BARE_SPECS+=("$2")
      shift 2
      ;;
    --help|-h)
      usage
      exit 0
      ;;
    *)
      echo "Unknown argument: $1" >&2
      usage
      exit 2
      ;;
  esac
done

if [[ -z "${USERS_CSV}" ]]; then
  echo "ERROR: --users is required" >&2
  usage
  exit 2
fi

if [[ ! -d .git ]]; then
  echo "ERROR: Run this from the root of the checked-out admin git repo." >&2
  exit 2
fi

REPO_ROOT="$(pwd)"
WORK_ROOT="${REPO_ROOT}/work"
BARE_ROOT="${WORK_ROOT}/bare"

IFS=',' read -r -a USERS <<< "${USERS_CSV}"

require_user_exists() {
  local user_name="$1"
  if ! id "${user_name}" >/dev/null 2>&1; then
    echo "Creating user: ${user_name}"
    useradd -m -s /bin/bash "${user_name}"
  else
    echo "User exists: ${user_name}"
  fi
}

ensure_group_exists() {
  if ! getent group "${GROUP_NAME}" >/dev/null; then
    echo "Creating group: ${GROUP_NAME}"
    groupadd "${GROUP_NAME}"
  else
    echo "Group exists: ${GROUP_NAME}"
  fi
}

ensure_user_in_group() {
  local user_name="$1"
  if id -nG "${user_name}" | tr ' ' '\n' | grep -qx "${GROUP_NAME}"; then
    echo "User ${user_name} already in ${GROUP_NAME}"
  else
    echo "Adding ${user_name} to ${GROUP_NAME}"
    usermod -a -G "${GROUP_NAME}" "${user_name}"
  fi
}

ensure_bashrc_umask() {
  local user_name="$1"
  local home_dir
  home_dir="$(getent passwd "${user_name}" | cut -d: -f6)"
  local bashrc="${home_dir}/.bashrc"

  touch "${bashrc}"
  chown "${user_name}:${user_name}" "${bashrc}"
  chmod 640 "${bashrc}"

  if ! grep -q '^[[:space:]]*umask[[:space:]]\+027[[:space:]]*$' "${bashrc}"; then
    cat >> "${bashrc}" <<'EOF'

# devgroup worker default: owner write, group read/search, others none
umask 027
EOF
  fi
}

ensure_private_home_dirs() {
  local user_name="$1"
  local home_dir
  home_dir="$(getent passwd "${user_name}" | cut -d: -f6)"

  chmod 750 "${home_dir}"
  install -d -m 700 -o "${user_name}" -g "${user_name}" "${home_dir}/.config"
  install -d -m 700 -o "${user_name}" -g "${user_name}" "${home_dir}/.cache"
  install -d -m 700 -o "${user_name}" -g "${user_name}" "${home_dir}/.local"
  install -d -m 700 -o "${user_name}" -g "${user_name}" "${home_dir}/tmp-private"
}

ensure_gitignore_has_work() {
  local gitignore="${REPO_ROOT}/.gitignore"
  touch "${gitignore}"

  if ! grep -qx 'work/' "${gitignore}"; then
    echo "Adding work/ to .gitignore"
    printf '\nwork/\n' >> "${gitignore}"
  fi
}

setup_shared_roots() {
  echo "Creating shared work roots"
  mkdir -p "${BARE_ROOT}"

  # Shared roots: mike/admin writable, group searchable/readable
  chown -R "${ADMIN_USER}:${GROUP_NAME}" "${WORK_ROOT}"
  chmod 2750 "${WORK_ROOT}"
  chmod 2750 "${BARE_ROOT}"

  # Default ACLs help new dirs/files under bare remain readable by group.
  if command -v setfacl >/dev/null 2>&1; then
    setfacl -m "u:${ADMIN_USER}:rwx,g:${GROUP_NAME}:rx" "${WORK_ROOT}" "${BARE_ROOT}" || true
    setfacl -d -m "u:${ADMIN_USER}:rwx,g:${GROUP_NAME}:rx" "${WORK_ROOT}" "${BARE_ROOT}" || true
  fi
}

setup_worker_tree() {
  local user_name="$1"
  local worker_root="${WORK_ROOT}/${user_name}"
  local github_root="${worker_root}/github"

  echo "Creating worker tree for ${user_name}: ${worker_root}"
  mkdir -p "${github_root}"

  # Worker owns only their subtree. Shared group can read/search.
  chown -R "${user_name}:${GROUP_NAME}" "${worker_root}"
  chmod 2750 "${worker_root}"
  chmod 2750 "${github_root}"

  if command -v setfacl >/dev/null 2>&1; then
    # Ensure mike/admin can recover/repair without sudo if needed.
    setfacl -m "u:${ADMIN_USER}:rwx,g:${GROUP_NAME}:rx" "${worker_root}" "${github_root}" || true
    setfacl -d -m "u:${ADMIN_USER}:rwx,g:${GROUP_NAME}:rx" "${worker_root}" "${github_root}" || true
  fi
}

init_bare_repo() {
  local spec="$1"
  local rel_path="${spec%%=*}"
  local upstream="${spec#*=}"

  if [[ "${rel_path}" == "${upstream}" ]]; then
    echo "ERROR: Invalid --init-bare spec: ${spec}" >&2
    exit 2
  fi

  local bare_path="${BARE_ROOT}/${rel_path}.git"
  local bare_parent
  bare_parent="$(dirname "${bare_path}")"

  mkdir -p "${bare_parent}"
  chown -R "${ADMIN_USER}:${GROUP_NAME}" "${BARE_ROOT}"

  if [[ -d "${bare_path}" ]]; then
    echo "Bare repo already exists: ${bare_path}"
    return 0
  fi

  echo "Initializing bare mirror: ${bare_path}"
  sudo -u "${ADMIN_USER}" git clone --bare "${upstream}" "${bare_path}"

  chown -R "${ADMIN_USER}:${GROUP_NAME}" "${bare_path}"

  # Directories readable/searchable by group; files readable by group.
  find "${bare_path}" -type d -exec chmod 2750 {} +
  find "${bare_path}" -type f -exec chmod 640 {} +

  if command -v setfacl >/dev/null 2>&1; then
    setfacl -R -m "u:${ADMIN_USER}:rwX,g:${GROUP_NAME}:rX" "${bare_path}" || true
    setfacl -R -d -m "u:${ADMIN_USER}:rwX,g:${GROUP_NAME}:rX" "${bare_path}" || true
  fi
}

  ensure_bashrc_umask "${ADMIN_USER}"
  ensure_private_home_dirs "${ADMIN_USER}"

  for user_name in "${USERS[@]}"; do
    require_user_exists "${user_name}"
    ensure_user_in_group "${user_name}"
    ensure_bashrc_umask "${user_name}"
    ensure_private_home_dirs "${user_name}"
  done

  ensure_gitignore_has_work
  mkdir -p "${WORK_ROOT}"
  setup_shared_roots

  for user_name in "${USERS[@]}"; do
    setup_worker_tree "${user_name}"
  done

  for spec in "${INIT_BARE_SPECS[@]}"; do
    init_bare_repo "${spec}"
  done

  print_next_steps
}

main "$@"

Why this should preserve shared readability
The important pieces are:

1. Worker subtree ownership
work/deva1 is owned by deva1:devgroup

2. Setgid on directories
chmod 2750 on worker dirs means new subdirs inherit group devgroup

3. umask 027
New files become group-readable, others denied

That combination is what makes worker-created clones readable/searchable by all group members without making them writable.

Remaining issues
A few matter.

1. Bare repos are not actually read-only at the Unix level unless you enforce it
In the script above, bare repos are writable by mike, readable by devgroup.

That is what you asked for, and it is right.

But if any worker ends up with ACL write permission on work/bare, that would be bad. So your validation script should explicitly test that workers cannot write under work/bare.

2. Existing sessions will not see new group membership immediately
Users already logged in may need:

new login shell

su - user

or logout/login

Otherwise group-based access can appear inconsistent.

3. Git safe.directory warnings
If mike or another user runs Git inside another user’s working tree, Git may complain about ownership.

That is normal.

Best practice:

the worker owns and uses their tree

mike imports from it in a separate writable integration clone

only add safe.directory selectively when needed

4. Model-provider auth is still a separate trust boundary
This setup solves cross-repo write control.

It does not solve:

what a worker agent can send over the network

whether source is exfiltrated to a provider

whether browser/API tokens inside a worker account are safe

That is outside the repo-permission model.

For now, keep provider creds under:

/home/devxx/.config/...

mode 600 or dir 700

and not under shared work/.

5. Filesystem type matters
This design assumes a Unix-permission-friendly filesystem under /mnt/disk2, such as:

ext4

xfs

btrfs

If this is NTFS, exFAT, or some unusual mount with weak ACL support, permissions may not behave as expected.

Check:

mount | grep ' /mnt/disk2 '
stat -f -c %T /mnt/disk2
6. Git clone permissions can still drift if tools override modes
Usually umask 027 + setgid dirs is enough. But some tooling may create extra files with different permissions.

That is why you want a validation and maybe a repair script.

7. Mike’s writable integration clones should stay separate
Do not use worker trees as the place where mike also edits and pushes.
Keep separate writable clones for mike under a different root.

That keeps:

worker proposal repos

integration repos

bare source mirrors

as three distinct roles.

8. Local bare clone vs bare mirror
The script currently uses:

git clone --bare
That is fine for worker local clone sources.

If you want a stricter mirror of upstream refs, you may later prefer:

git clone --mirror
Difference:

--bare is enough for ordinary clone-source usage

--mirror mirrors all refs and is better if Mike is using these as upstream-ref caches

For your current use, either is fine. I would lean --mirror later if these are true maintenance mirrors.

What I would add next
You asked about remaining issues. The biggest missing piece after setup is validation.

You should have a companion script that checks:

each worker can write only under their own work/devxx

other workers can read but not write that subtree

workers cannot write under work/bare

bare repos are cloneable

new clone contents are group-readable

mike can fetch from worker trees

That is the next thing I would implement.

Thought for 6s
Here is a companion validation script for the setup. It checks:

users and group exist

key directories exist and have sane ownership/modes

each worker can write in their own subtree

another worker can read but not write that subtree

workers cannot write under work/bare

workers can clone from a local bare repo

cloned files come out group-readable

Save this as:

scripts/validate-devgroup.sh
and run it with sudo from the root of your admin repo.

scripts/validate-devgroup.sh
#!/usr/bin/env bash
set -euo pipefail

GROUP_NAME="devgroup"
ADMIN_USER="mike"
USERS_CSV=""
TEST_BARE_REL=""   # example: sw-embed/sw-cor24-apl.git

usage() {
  cat <<EOF
Usage:
  sudo $0 --users user1,user2,... [--group devgroup] [--admin mike]
           [--test-bare org/repo.git]

Options:
  --group      Shared unix group (default: devgroup)
  --admin      Admin/coordinator user (default: mike)
  --users      Comma-separated worker users (required)
  --test-bare  Relative path under work/bare to use for clone test
               Example: sw-embed/sw-cor24-apl.git
  --help       Show this help

Examples:
  sudo $0 --users deva1,deva2,devb1
  sudo $0 --users deva1,deva2 --test-bare sw-embed/sw-cor24-apl.git
EOF
}

while [[ $# -gt 0 ]]; do
  case "$1" in
    --group)
      GROUP_NAME="$2"
      shift 2
      ;;
    --admin)
      ADMIN_USER="$2"
      shift 2
      ;;
    --users)
      USERS_CSV="$2"
      shift 2
      ;;
    --test-bare)
      TEST_BARE_REL="$2"
      shift 2
      ;;
    --help|-h)
      usage
      exit 0
      ;;
    *)
      echo "Unknown argument: $1" >&2
      usage
      exit 2
      ;;
  esac
done

if [[ -z "${USERS_CSV}" ]]; then
  echo "ERROR: --users is required" >&2
  usage
  exit 2
fi

if [[ ! -d .git ]]; then
  echo "ERROR: Run this from the root of the checked-out admin repo." >&2
  exit 2
fi

REPO_ROOT="$(pwd)"
WORK_ROOT="${REPO_ROOT}/work"
BARE_ROOT="${WORK_ROOT}/bare"

IFS=',' read -r -a USERS <<< "${USERS_CSV}"

FAILURES=0
WARNINGS=0

pass() { echo "PASS: $*"; }
warn() { echo "WARN: $*"; WARNINGS=$((WARNINGS + 1)); }
fail() { echo "FAIL: $*"; FAILURES=$((FAILURES + 1)); }

check_exists() {
  local path="$1"
  [[ -e "$path" ]]
}

check_user() {
  local u="$1"
  if id "$u" >/dev/null 2>&1; then
    pass "user exists: $u"
  else
    fail "user missing: $u"
  fi
}

check_group() {
  local g="$1"
  if getent group "$g" >/dev/null 2>&1; then
    pass "group exists: $g"
  else
    fail "group missing: $g"
  fi
}

check_user_in_group() {
  local u="$1"
  local g="$2"
  if id -nG "$u" | tr ' ' '\n' | grep -qx "$g"; then
    pass "user $u is in group $g"
  else
    fail "user $u is NOT in group $g"
  fi
}

show_stat() {
  local path="$1"
  stat -c '%A %a %U:%G %n' "$path"
}

check_path() {
  local label="$1"
  local path="$2"
  if [[ -e "$path" ]]; then
    pass "$label exists: $path"
    show_stat "$path"
  else
    fail "$label missing: $path"
  fi
}

check_bashrc_umask() {
  local u="$1"
  local home
  home="$(getent passwd "$u" | cut -d: -f6)"
  local bashrc="${home}/.bashrc"

  if [[ ! -f "$bashrc" ]]; then
    fail "$u .bashrc missing: $bashrc"
    return
  fi

  if grep -q '^[[:space:]]*umask[[:space:]]\+027[[:space:]]*$' "$bashrc"; then
    pass "$u .bashrc contains umask 027"
  else
    warn "$u .bashrc missing explicit 'umask 027'"
  fi
}

write_test_own_tree() {
  local u="$1"
  local probe_dir="${WORK_ROOT}/${u}/github/.validate"
  local probe_file="${probe_dir}/own-write.txt"

  if sudo -u "$u" mkdir -p "$probe_dir" &&
     sudo -u "$u" bash -lc "umask 027; echo ok > '$probe_file'"; then
    pass "$u can write in own tree"
  else
    fail "$u cannot write in own tree"
    return
  fi

  if [[ -f "$probe_file" ]]; then
    show_stat "$probe_file"
    local mode
    mode="$(stat -c '%a' "$probe_file")"
    case "$mode" in
      640|660|600)
        pass "$u probe file has plausible mode: $mode"
        ;;
      *)
        warn "$u probe file mode unusual: $mode"
        ;;
    esac
  else
    fail "$u probe file missing after write"
  fi
}

cross_read_write_test() {
  local owner="$1"
  local other="$2"

  local probe_dir="${WORK_ROOT}/${owner}/github/.validate"
  local probe_file="${probe_dir}/own-write.txt"
  local denied_file="${probe_dir}/cross-write-should-fail.txt"

  if sudo -u "$other" test -r "$probe_file"; then
    pass "$other can read $owner shared file"
  else
    fail "$other cannot read $owner shared file"
  fi

  if sudo -u "$other" bash -lc "echo nope > '$denied_file'" 2>/dev/null; then
    fail "$other WAS able to write into $owner tree"
  else
    pass "$other cannot write into $owner tree"
  fi
}

bare_write_denial_test() {
  local u="$1"
  local probe_dir="${BARE_ROOT}/.validate"
  mkdir -p "$probe_dir"
  chown "${ADMIN_USER}:${GROUP_NAME}" "$probe_dir"
  chmod 2750 "$probe_dir"

  local denied_file="${probe_dir}/${u}-should-fail.txt"

  if sudo -u "$u" bash -lc "echo nope > '$denied_file'" 2>/dev/null; then
    fail "$u WAS able to write under work/bare"
  else
    pass "$u cannot write under work/bare"
  fi
}

clone_test() {
  local u="$1"
  local rel="$2"
  local src="${BARE_ROOT}/${rel}"
  local clone_root="${WORK_ROOT}/${u}/github/.validate-clones"
  local dest="${clone_root}/$(basename "$rel" .git)"

  if [[ ! -d "$src" ]]; then
    fail "clone test source bare repo missing: $src"
    return
  fi

  sudo -u "$u" mkdir -p "$clone_root"
  sudo -u "$u" rm -rf "$dest"

  if sudo -u "$u" git clone "$src" "$dest" >/dev/null 2>&1; then
    pass "$u can clone from local bare repo: $rel"
  else
    fail "$u failed to clone from local bare repo: $rel"
    return
  fi

  check_path "clone destination" "$dest"

  local sample_file
  sample_file="$(find "$dest" -type f | head -n 1 || true)"
  if [[ -n "$sample_file" ]]; then
    show_stat "$sample_file"
    if [[ -r "$sample_file" ]]; then
      pass "sample cloned file exists"
    fi
  else
    warn "no sample file found in cloned repo: $dest"
  fi

  # Check another worker can read but not write this clone.
  local other=""
  for cand in "${USERS[@]}"; do
    if [[ "$cand" != "$u" ]]; then
      other="$cand"
      break
    fi
  done

  if [[ -n "$other" ]]; then
    if sudo -u "$other" test -r "$dest/.git/config"; then
      pass "$other can read $u cloned repo metadata"
    else
      fail "$other cannot read $u cloned repo metadata"
    fi

    if sudo -u "$other" bash -lc "echo nope > '$dest/WRITE_SHOULD_FAIL.txt'" 2>/dev/null; then
      fail "$other WAS able to write into $u cloned repo"
    else
      pass "$other cannot write into $u cloned repo"
    fi
  fi
}

main() {
  echo "Repo root:  $REPO_ROOT"
  echo "Work root:  $WORK_ROOT"
  echo "Bare root:  $BARE_ROOT"
  echo

  check_group "$GROUP_NAME"
  check_user "$ADMIN_USER"
  check_user_in_group "$ADMIN_USER" "$GROUP_NAME"
  echo

  check_path "work root" "$WORK_ROOT"
  check_path "bare root" "$BARE_ROOT"
  echo

  for u in "${USERS[@]}"; do
    check_user "$u"
    check_user_in_group "$u" "$GROUP_NAME"

    local_home="$(getent passwd "$u" | cut -d: -f6)"
    check_path "home for $u" "$local_home"
    check_bashrc_umask "$u"

    check_path "worker root for $u" "${WORK_ROOT}/${u}"
    check_path "worker github root for $u" "${WORK_ROOT}/${u}/github"
    echo
  done

  for u in "${USERS[@]}"; do
    write_test_own_tree "$u"
  done
  echo

  if [[ ${#USERS[@]} -ge 2 ]]; then
    for ((i=0; i<${#USERS[@]}; i++)); do
      owner="${USERS[$i]}"
      other="${USERS[$(( (i + 1) % ${#USERS[@]} ))]}"
      if [[ "$owner" != "$other" ]]; then
        cross_read_write_test "$owner" "$other"
      fi
    done
    echo
  else
    warn "cross-user tests skipped; need at least 2 users"
    echo
  fi

  for u in "${USERS[@]}"; do
    bare_write_denial_test "$u"
  done
  echo

  if [[ -n "$TEST_BARE_REL" ]]; then
    clone_test "${USERS[0]}" "$TEST_BARE_REL"
    echo
  else
    warn "clone test skipped; pass --test-bare org/repo.git to enable it"
    echo
  fi

  echo "Validation summary:"
  echo "  failures: ${FAILURES}"
  echo "  warnings: ${WARNINGS}"

  if [[ "$FAILURES" -ne 0 ]]; then
    exit 1
  fi
}

main "$@"
